关于MySQL导入数据库报错

MySQL数据库导入SQL报错 Unknown collation: ‘utf8mb4_0900_ai_ci’

从8.0以上版本MySQL导出的sql文件字符集为utf8mb4_0900_ai_ci，与5.7版本的utf8_general_ci不兼容，打开sql文件把所有COLLATE都改成utf8_general_ci即可解决。

静态变量和方法

类变量

类变量就是静态变量(static)
最大的特点：会被所在类所有对象实例贡献。

类变量是该类的所有对象共享的变量，任何一个该类的对象访问它时，取到的都是相同的值，同样的任何一个该类的对象修改变量时，修改的也都是同一变量。

类变量的使用注意事项和细节

  1. 适用的场景：
    当需要让某个类的所有对象共享一个变量时，就可以考虑使用类变量，如：定义学生类，统计所有学生的交费总数。
  2. 类变量与实例变量(普通属性)的区别
    类变量是该类的所有对象共享的，而实例变量是每个对象独享的。
  3. 不加static的是实例变量/普通变量/非静态变量。
  4. 类变量可以通过类名.类变量名或者对象名.类变量名访问，但是java设计者推荐使用前者
  5. 实例变量不能通过类名.类变量名来访问
  6. 类变量是在类加载时就初始化的，也就是说，即使没有创建对象，只要类加载了就可以使用类变量了。
  7. 类变量的生命周期是随类加载开始，随类消亡而销毁

类方法

创建和调用都和类方法相同

类方法的经典使用场景：
当方法中不涉及到任何和对象相关的成员，则可以将方法设计为静态方法，提高开发效率。如：各种工具类。

在类方法中只能访问静态方法和静态变量
普通方法中则既可以访问普通成员也可以访问静态成员

深入理解main方法

  1. main方法是java虚拟机调用的

  2. java虚拟机需要调用main()方法，所以该方法的访问权限必须是public

  3. java虚拟机在执行main()方法时不必创造对象，所以该方法必须是static的

  4. 该方法接收String类型的数组参数，该数组中保存执行java命令时传递给所运行的类的参数

特别提示

  1. 在main()方法中，可以直接调用main()方法所在类的静态方法或者属性
  2. 但是不能直接访问该类中的非静态成员，需要先实例对象。

代码块又称作初始化块，属于类中的成员，类似于方法，是将逻辑语句用{ }包围起来
但是和方法不同，代码块没有方法名、返回类型和参数，只有方法体，而且不能通过对象和类名显示调用，是在加载类时或者创建对象时隐式调用。

  1. 修饰符写的话只能写static
  2. 代码块分为两类，有static的是静态代码块，没有的是普通代码块。
  3. ;可以省略

如果类中的多个构造器，都有同样的初始代码时我们可以把初始条件放入代码块中，减少冗余。

代码块的注意事项和细节

  1. static代码块也叫静态代码块，作用是对类进行初始化，它随着类的加载而执行，但只会执行一次，普通代码块则每创建一个对象都会执行一次。
  2. 类加载的时机（很重要）
    1）创建对象实例时(new)
    2）创建子类对象实例时，父类也会被加载。
    （ 因为加载子类时，必须要先加载父类）
    3）实用类的静态成员时
  3. 普通代码块，在创建对象实例时，会被隐式调用。
    如果只是使用类的静态成员时，普通代码块则不会执行。
  4. 创建一个对象时，在一个类调用顺序（很重要）
    1）调用静态代码块和静态属性初始化（静态代码块和静态属性初始化调用的优先级一样，如果多个静态代码块和多个静态变量初始化，则按他们定义的顺序调用）
    2）调用普通代码块和普通属性的初始化（普通代码块和普通属性初始化调用优先级一样，如果多个普通代码块和多个普通属性初始化，则按定义顺序调用）
    3）调用构造方法
  5. 构造器的最前面隐含了super()和调用普通代码块。
    调用子类构造器 会先调用父类构造器
    再调用子类普通代码块
    最后是子类构造器
  6. 创建一个子类对象时，静态代码块、静态属性初始化、普通代码块、普通属性初始化和构造函数的调用顺序如下。
    1）父类的静态代码块和静态属性(优先级相同，按定义顺序执行)
    2）子类的静态代码块和静态属性(优先级相同，按定义顺序执行)
    3）父类的普通代码块和普通属性(优先级相同，按定义顺序执行)
    4）父类的构造方法
    5）子类的普通代码块和普通属性(优先级相同，按定义顺序执行)
    6）子类的构造方法

单例设计模式

是静态方法和属性的经典使用场景

  1. 所谓单例设计模式，就是采取一定的方法保证在整个软件系统中，对某个类只能存在一个对象实例，并且该类只提供一个取得其对象的方法。
  2. 单例模式有两种方式饿汉式(初始化类就实例化对象)和懒汉式(调用对象时才创建)

“饿汉式”单例
这个没有线程安全的问题,但是会造成资源的浪费

  1. 把构造器private
    2.在类的内部创建对象（private static）
  2. 提供一个public static的方法(getInstance())，返回创建的对象

“懒汉式”单例
这个有线程安全的风险，但是不会造成资源的浪费

  1. 把构造器private
  2. 定义一个static的属性对象
  3. 提供一个public的static方法，返回一个对象实例。

区别：

  1. 二者的最主要区别是创建对象的时机不同：饿汉是在类加载就创建了对象实例而懒汉是在使用时才创建。
  2. 饿汉不存在线程安全问题，懒汉存在线程安全问题（后续会有完善的方式）
  3. 饿汉存在浪费资源的可能。因为如果程序员没有使用对象实例，创建的对象会被浪费；而懒汉因为使用时才创建，所以不存在这个问题
  4. java.lang.Runtime就是经典的单例模式。
