排序

排序时将多个数据依指定的顺序进行排列的过程。

分为：

  1. 内部排序
    指将需要处理的所有数据都加载到内部储存器中进行排序。（包括交换式排序法、选择式排序法和插式排序法）

  2. 外部排序
    数据量过大无法全部加载到内存中时，需要借助外部存储进行排序。（包括合并排序法和直接合并排序法）

这里只学习一个冒泡排序(Bubble Sorting)，后续的会在算法部分学习。

{24，69，80，57，13}
分析冒泡排序：

第一轮排序：目标把最大的数放在最后
第一次比较 24比69 ，前小后大不变位置。
第二次比较69：80 前小后大不变位置。
第三次比较 80：57，80较大，所以80，57换位 {24，69，57，80，13}
第四次比较 80：13，80较大，换位 {24，69，57，13，80} 第一轮排序结束

第二轮排序：目标把次大的，放在length-2的位置
第一次比较24：69，前小后大不变位置。
第二次比较69：57，换位 {24,57,69,13,80}
第三次比较69：13，换位{24,57,13,69,80}
69 和80没有再比较的意义，所以不进行第四轮比较

第三轮排序：目标把第三大的数，放在length-3的位置
第一次比较24：57，不变
第二次比较57：13，交换{24,13,57,69,80}
后面都是比较过的了 本轮排序结束。

最后一轮排序：确定最后两个数的位置。
第一次比较24：13,交换{13,24,57,69,80}
后面都是排序过的数，无需比较

特点总结：

  1. 有n个元素的array，则进行n-1轮排序。(外层循环)
  2. 每轮排序可以确定一个元素的位置。
  3. 当进行比较时，若前元素大于后元素则交换位置。
  4. 每轮的比较次数，逐渐减少(arr1.length - 1 - i)(内层循环)

switch分支结构

  1. switch关键字，表示switch分支。

  2. 表达式对应一个值。

  3. 当表达式的值等于常量1时，就执行语句块1。

  4. break;表示退出switch代码块。

  5. 如果case 常量1不匹配就继续向下匹配case 常量2，以此类推。

switch分支细节

  1. 表达式数据类型应和case后的常量类型一致，或者是可以自动转换成可以相互比较的类型，比如输入的是char，而常量是int。

  3. case后面跟的必须是常量或者常量表达式，不能是变量。

  4. default子句是可选的。

switch和if的选择

  1. 如果判断的具体数值不多，而且符合byte、short、int、char、enum、String类型，虽然两种都能用，但是推荐使用switch。

  2. 对区间判断，对结果为boolean类型的判断，用if-else较好。

位运算符

二进制（Binary）在运算中的说明

  1. 二进制是逢2进位制，0、1是基本算符。

  2. 现代的电子计算机技术全部采用的二进制，因为只使用0、1两个算符，非常简洁方便，易于用电子方式实现。

原码、反码、补码（重点 要背过）

  1. 二进制的最高位是符号位：0表示正数，1表示负数。

  2. 正数的原码、反码、补码都一样。（三码合一）

  3. 负数的反码 = 原码符号位不变，其他位取反。

  4. 负数的补码 = 反码 + 1，负数的反码 = 补码 - 1。

  5. 0的反码和补码都是0。

  6. Java没有无符号数，换言之Java中得数都是有符号的。

  7. 计算机运算得时候都是以补码得方式进行的。（所以自己算要先反 → 补 → 算 → 反 → 原）

  8. 看运算结果，要看原码。

位运算符

Java中有七个位运算符（&、|、^、~、>>、<<、>>>）

分两组归纳：

第一组：

    11011110

因为1是正数，反码补码都和原码相同，故补码亦如上。

反码：11111111 11111111 11111111 11111101
补码：11111111 11111111 11111111 11111110

00000000 00000000 00000000 00000001

3. 按位异或(^)；两位一个为0一个为1时，则结果为1，否则为0.

  1. 按位取反(~)： 0 -> 1, 1 -> 0

第二组：

  1. 算术右移(>>)：低位溢出（就是舍弃的意思），符号位不变，并用符号位补齐溢出的高位。

标识符命名规则与规范

  1. Java对各种变量、方法和类的命名时使用的字符序列称为标识符

  2. 凡是自己可以起名字的地方都叫标识符

命名规则（若违反则编译报错）

  1. 由26个英文字母大小写，0-9、$和_组成。

  2. 数字不能作为开头

  3. 不可以使用关键字和保留字，但是可以包含关键字和保留字。

  4. Java中严格区分大小写，长度无限制。

  5. 不能包含空格

命名规范（若违背不会编译报错，但是应当遵守）

  1. 包名：多单词组成时所有字母都小写,例 aaa.bbb.ccc

  2. 类名、接口名：由多个字母组成时所有单词的首字母都大写，例：AaaBbbCbb

  3. 变量名、方法名：由多单词组成时，第一个单词首字母小写，从第二个单词开始首字母大写，例：aaaBbbCcc

  4. 常量名；所有字母都大写，多单词时每个单词之间用_连接，例：AAA_BBB_CCC

保留字：现在的Java版本中尚未使用，但是以后版本可能会作为关键字使用。

  4. 十六进制：0-9及1（10）-F（15），满16进1，以0X或0x开头，A-F不区分大小写。例：0x10101

进制转换（基本功） 特别重要

第一组：

  1. 2转10
    规则：从最低位开始，将每个位上的数提出来 ，乘以2的（位数-1）次方，并求和。

  2. 8转10
    规则：从最低位开始，将每个位上的数提出来，乘以8的（位数-1）次方，并求和。

  3. 16转10
    规则：从最低位开始，将每个位上的数提出来，乘以16 的（位数-1）次方，并求和。

第二组：

  1. 10转2
    规则：将该数不断的除以2，直到商除不尽为止，然后将每步得到的余数倒过来就是对应的二进制。（2转10的逆过程）

  2. 10转8
    规则：将该数不断的除以8，直到商除不尽为止，然后将每步得到的余数倒过来就是对应的二进制。（8转10的逆过程）

  3. 10转16
    规则：将该数不断的除以16，直到商除不尽为止，然后将每步得到的余数倒过来就是对应的二进制。（16转10的逆过程）

例：
十进制数34转2进制 => 0b00100010 （因为一个字节有8位，不够的需要0填位）

第三组：

  1. 2转8
    规则：从低位开始，将二进制数每三位一组，转成对应的八进制数即可。

  2. 2转16
    规则：从低位开始，将二进制数每四位一组，转成对应的八进制数即可。

第四组：

  1. 8转2
    规则：将八进制数每一位，转成对应的一个三位的二进制数即可。（2转8的逆过程）

  2. 16转2
    规则：将八进制数每一位，转成对应的一个四位的二进制数即可。（2转16的逆过程）

2024-3-02学习笔记

今天的内容是运算符

关系运算符（比较运算符）

关系运算符的结果是boolean类型，只能是true和false

关系表达式经常用在if结构的条件或者循环结构的条件中

==、!=、<、>、<=、>=、instanceof(检查是否是类的对象，到面向对象时会细讲)

关系运算符组成的表达式，称为关系表达式

逻辑运算符

用于连接多的条件（多个关系表达式），最终结果也是一个boolean值

&、&&、|、||、!、^（逻辑异或）

逻辑运算符规则
a & b:&叫逻辑与 a和b同时为true则结果为true否则为false。
a && b：短路与 a和b同时为true则结果为true否则为false。

a | b：逻辑或 当a和b中有一个为true则结果为true否则为false。
a || b：短路或 当a和b中有一个为true则结果为true否则为false。
!a:取反或者叫非运算。当a为true时结果为false，反之亦然。

a ^ b：逻辑异或，当a和b不同时结果为true否则为false。

&和&&的使用区别

  1. &&：如果第一个条件为false，则最终结果直接为false第二个条件不会判断，效率高。

  2. &：不管第一个条件是否为false，第二个都要判断，效率低。

  3. 开发中基本都是使用&&

|和||的使用区别

  1. ||：如果第一个条件为true，则最终结果直接为true第二个条件不会判断，效率高。

  2. |：不管第一个条件是否为true，第二个都要判断，效率低。

  3. 开发中基本都是使用||。

逻辑非与逻辑异或

!a:逻辑非或者逻辑取反，若a为true则!a为false，反之亦然。

a^b：逻辑异或，a和b的结果不同时，a^b为true；a和b结果相同时a^b为false。

一个练习(读代码分析过程和输出结果)

因为过程会有点长，在下面分析：
if((z++ == 46)&&(y = true))z++;
//z++ == 46,先比较后自增，所以为true，z为47；因为时&&程序进关系表达式2，y = true 注意这里是=赋值不是==，所以也是true。条件符合进入代码体 z++，z为48
if((x = false)||(++z == 49))z++;
//同样第一部分是赋值=不是==，x为false因为是||所以要判断条件2，++z == 49先自增后比较z为49且条件为true，进入代码体z++，z为50.
所以最终输出为50。

赋值运算符

将某个运算后的值，赋给指定变量，除了=之外还有+=、-=、*=、=

赋值运算符的特点

  1. 运算顺序从右往左（先运算完右边的整体再赋值给左边变量）

  2. 赋值运算符的左边只能是一个变量，右边可以是变量、表达式或者常量。

  3. 复合赋值运算符等价于以下效果
    a += 3; a = a + 3;

  4. 复合赋值运算符会进行类型转换

但是

byte b = b + 2; //这个会报错，因为右边是int型无法赋值给byte类变量b

b++; //底层也是包含一个强类型转换的

三元运算符

基本语法：
条件表达式 ? 表达式1 : 表达式2;

规则；

  1. 如果条件表达式为true，运算后的结果是表达式1。

  2. 如果条件表达式为false，运算后的结果是表达式2。

三元运算符细节

  1. 表达式1和表达式2要是可以赋值给左边变量的类型，或者可以自动转换的。

  2. 三元运算符可以转换为if else语句
