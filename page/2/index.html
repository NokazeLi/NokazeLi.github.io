super关键字

super代表父类的引用，用于访问父类的属性、方法和构造器。

  1. 访问属性和方法，但不能访问private的属性和方法。
  2. 访问父类的构造器，只能放在构造器的第一句且只能写一句。

super的好处和细节

  1. 好处:分工明确，父类属性由父类初始化，子类属性由子类初始化。
  2. 当子类中有和父类中的属性和方法重名重名的属性和方法时，为了访问父类成员(属性和方法)，必须使用super。若没有重名，使用super、this和默认访问效果都相同。
  3. super的访问不限于直接父类，如果父类的父类和本类中有同名的成员，也可以使用super去访问其中的成员;若多级父类中都有同名成员，使用super访问则遵循就近原则。(调用最近的父类关系)

方法重写/覆盖(override)

子类有一个方法和父类的某一个方法名称、返回类型、形参列表一样，则子类方法覆盖(重写)了父类方法。

注意事项和细节

  1. 子类方法的参数、方法名，要和父类方法完全一样。
  2. 子类方法的返回类型和父类方法返回类型一样，或者是父类返回类型的子类，比如：父类返回类型时Object子类方法的返回类型是String。
  3. 子类方法不能缩小父类的访问权限。

方法活对象具有多种形态。是面向对象的第三大特征，多态是建立在封装和继承基础之上的。

向上转型

  1. 本质：父类的引用只想子类的对象。
  2. 语法：父类类型 引用名 = new 子类类型();
  3. 特点：可以调用父类中的所有成员(需要遵守访问权限)，不能调用子类的特有成员；最终运行效果看子类的具体表现。

线上转型的调用规则和继承的调用规则相同，先从当前运行类型开始查找，若运行类型中有调用的成员则调用。否则逐级向上查找，直到找到父类中相应的成员。

向下转型

向上转型只能调用父类中的成员，而不能调用子类的特有成员。所以通过向下转型调用子类特有成员。

  1. 语法：子类类型 引用名 = (子类类型)父类引用； //强转
  2. 只能强转父类的引用，而不能强转父类的对象。 //比如 Animal之于Cat，只能将Animal引用的Cat对象强转Cat，而不能将Animal对象转Cat。
  3. 要求父类的引用必须只想当前目标类型的对象。//如果运行类型是cat的对象，也只能强转cat而不能强转dog。
  4. 可以调用子类类型中的所有成员。 //完成了对向上转型的功能补充

访问修饰符

java共提供了四种访问修饰符，用于控制方法和属性（成员变量）的访问权限。

  1. 公开级别:public 对外公开。
  2. 受保护级别:protected，对子类和同一个包中的类公开。
  3. 默认：啥都不写的，向同一个包的类公开。
  4. 私有级别：private，只有类本身可以访问，不对外公开

注意事项

  1. 修饰符可以用来修饰类中的属性、成员方法以及类。
  2. 只有默认的和public才可以修饰类。

封装

封装(encapsulation)就是把抽象出的数据(属性)和对数据的操作(方法)封装在一起,数据被保护在内部。程序的其他部分只有通过被授权的操作(方法)才能对数据进行操作。

封装的好处

  1. 隐藏实现细节
  2. 可以对数据进行验证,保证安全合理

实现步骤

  1. 先将属性私有化 private。
  2. 提供一个public的set方法用于对属性判断和赋值。
  3. 提供一个public的get方法用于获取属性的值。

继承

继承可以解决代码复用，让编程更接近人类思维。当多个类存在相同的属性和方法时，可以从这些类中抽象出父类，在父类中定义这些相同的属性和方法，所有的子类不需要重新定义这些属性和方法，只需要通过extends来声明继承父类即可。

语法

子类会自动拥有父类的属性和方法
父类也叫超类、基类。
子类也叫派生类。
使用继承提高了代码的复用性，同时也提高了代码的扩展性和维护性。

注意事项和细节

  1. private的属性和方法不能在直接访问，只能通过父类提供的public方法访问（比如get方法）。
  2. 子类必须调用父类的构造器，完成父类的初始化。
  3. 当创建子类对象时，不管使用子类的哪个构造器，默认情况下总会去调用父类的无参构造器，如果父类没有提供无参构造器，则必须在子类的构造器中用super去指定使用父类的哪个构造器完成对父类的初始化工作，否则编译不通过。
  4. 如果希望指定去调用父类的某个构造器，则显式调用一下super(参数列表)。
  5. super在使用时，需要放在构造器第一行。(super只能在构造器里调用)
  6. super()和this()都只能放在构造器第一行，因此这两个方法不能共存在同一个构造器中。
  7. Object类是所有类的基类
  8. 父类构造器的调用不限于直接父类，将一直向上追溯至Object类。
  9. 子类最多技能一个父类(指直接继承)，即java是单继承机制。
  10. 继承不可滥用，子类和父类之间关系必须满足is-a的逻辑关系(子类is a 父类)

    局部变量不可以加修饰符

方法递归调用

递归就是方法调用其本身，每次调用时传入不同的变量。递归有助于编程者解决复杂的问题使代码变简洁。

递归能解决的问题：

  1. 各种数学问题，如：八皇后问题、汉诺塔问题、阶乘问题、迷宫问题、球和篮子问题等。

  2. 各种算法中也使用递归，比如快排、归并排序、二分查找、分治算法等。

递归规则：

  1. 执行一个方法时，就闯进一个新的受保护的独立空间(栈空间)。

  2. 方法的局部变量是独立的不会互相受影响。

  3. 如果方法中使用的引用类型(比如数组)，就会共享该引用类型的数据。

  4. 递归必须向对出递归的条件逼近，否则回无限递归(StackOverFlowError)。

  5. 当一个方法执行完毕，或者遇到return就会返回，遵循谁调用就把结果返回给谁的规律，方法执行完毕返回时，该方法就结束了。

  2. 参数类型可以为基本类型和引用类型。

  3. 调用带参数的方法时，一定要传入相同类型或者兼容的类型。

  4. 方法定义时的参数称为形式参数(形参)；方法调用时的参数称为实际参数(实参)。形参和实参的类型要一致或者兼容，个数和顺序必须一致。

不可以在方法中 再定义另一个方法。

方法调用的细节

  1. 在同一个类中的方法调用:直接调用即可。

  2. 跨类调用时，要先创建对象。

方法的传参机制(极其重要)

  1. 对于基本类型,传递的时值(值copy)，形参的任何改变不影响实参。

  2. 对于引用类型则是地址copy，形参的改变同样影响实参。

排序

排序时将多个数据依指定的顺序进行排列的过程。

分为：

  1. 内部排序
    指将需要处理的所有数据都加载到内部储存器中进行排序。（包括交换式排序法、选择式排序法和插式排序法）

  2. 外部排序
    数据量过大无法全部加载到内存中时，需要借助外部存储进行排序。（包括合并排序法和直接合并排序法）

这里只学习一个冒泡排序(Bubble Sorting)，后续的会在算法部分学习。

{24，69，80，57，13}
分析冒泡排序：

第一轮排序：目标把最大的数放在最后
第一次比较 24比69 ，前小后大不变位置。
第二次比较69：80 前小后大不变位置。
第三次比较 80：57，80较大，所以80，57换位 {24，69，57，80，13}
第四次比较 80：13，80较大，换位 {24，69，57，13，80} 第一轮排序结束

第二轮排序：目标把次大的，放在length-2的位置
第一次比较24：69，前小后大不变位置。
第二次比较69：57，换位 {24,57,69,13,80}
第三次比较69：13，换位{24,57,13,69,80}
69 和80没有再比较的意义，所以不进行第四轮比较

第三轮排序：目标把第三大的数，放在length-3的位置
第一次比较24：57，不变
第二次比较57：13，交换{24,13,57,69,80}
后面都是比较过的了 本轮排序结束。

最后一轮排序：确定最后两个数的位置。
第一次比较24：13,交换{13,24,57,69,80}
后面都是排序过的数，无需比较

特点总结：

  1. 有n个元素的array，则进行n-1轮排序。(外层循环)
  2. 每轮排序可以确定一个元素的位置。
  3. 当进行比较时，若前元素大于后元素则交换位置。
  4. 每轮的比较次数，逐渐减少(arr1.length - 1 - i)(内层循环)

嵌套分支

因为可读性太差，推荐嵌套不要超过三层

switch分支结构

  1. switch关键字，表示switch分支。

  2. 表达式对应一个值。

  3. 当表达式的值等于常量1时，就执行语句块1。

  4. break;表示退出switch代码块。

  5. 如果case 常量1不匹配就继续向下匹配case 常量2，以此类推。

  6. 如果都没匹配成功，则执行default。

没有break;代码会向下穿透

switch分支细节

  1. 表达式数据类型应和case后的常量类型一致，或者是可以自动转换成可以相互比较的类型，比如输入的是char，而常量是int。

  2. switch的表达式的返回值必须是：byte、short、int、char、enum、String。

  3. case后面跟的必须是常量或者常量表达式，不能是变量。

  4. default子句是可选的。

switch和if的选择

  1. 如果判断的具体数值不多，而且符合byte、short、int、char、enum、String类型，虽然两种都能用，但是推荐使用switch。

  2. 对区间判断，对结果为boolean类型的判断，用if-else较好。
