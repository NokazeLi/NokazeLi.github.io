2024-2-29学习笔记

28号学习了基础类型中的数值类，今天学习了剩下的。

字符类型 char

char可以存储单个字符

做个字符用String类存储

char有两个字节可以存储单个汉字

字符类型细节

  1. 字符常量用单引号括起来，不可以用双引号 双引号括起来的是字符串

  2. char可以储存转义字符

  3. 在Java中char的本质是一个整数，输出的是Unicode码对应的字符

  4. 给char类型赋一个数值，会输出对应的Unicode码代表的字符
    例：
    char c1 =97；
    System.out.println(c1); //输出a

    也可以反向输出，但是要用强转类型

    char c2 =’a’;
    System.out.println(c1); //输出97

  5. 因为char的本质是一个整数，所以可以参与计算
    例：
    System.out.println(‘a’ + 10); //输出97

具体数值代表的字符可以查询ASCII表

字符类型的本质

  1. 字符类型存储到计算机中，需要把字符对应的码值找出来
    比如：
    存储：’a’ ==> 码值97 ==> 二进制 ==> 存储
    读取: 二进制 ==> 97 ==> ‘a’ ==> 显示

  2. 字符和码值的顺序是通过字符编码表规定的

常用的字符编码

ASCII表；1个字节表示，一共有128个字符（实际一个字节至多可以表示256个字符）ASCII是上世纪六十年代美国指定的一套字符编码，对英语字符与二进制位之间的关系做了统一规定，共规定了128个字符的编码，值占用了1个字节的后七位，最前面一位统一规定为0
缺点：只能表示英文字母

Unicode：Unicode编码表，固定大小的编码，字母和汉字统一占用2个字符。Unicode是一种将世界上所有符号都纳入其中的编码，每一个符号都给予了一个独一无二的编码，使用Unicode编码不存在乱码问题.
缺点:单个英文字母和汉字都占用2个字节，比较浪费存储空间。
Unicode编码的0-127与ASCII相同，所以Unicode兼容ASCII

UTF-8：大小可变的编码，字母使用1个字节而汉字使用3个字节，所以UTF-8能存储比gbk更多的汉字.UTF-8是互联网上使用最广的一种Unicode的实现方式。是一种变长的编码方式，它可以使用1-6字节表示一个符号，会根据不同的符号而变化字节长度。

gbk:可以表示汉字而且范围广，字母使用1个字节汉字使用2个字节存储

gb2312:可以表示汉字，表示的汉字较少 gb2312 < gbk

big5:繁体中文

布尔型

  1. boolean只允许取值true和false，不能是null。

  2. boolean只占1个字符。

  3. 在Java里不可以用0和非0代替false和true。

基本数据类型自动转换

当Java程序在进行赋值或者运算时，精度小的类型自动转换为精度大的数据类型，这就是自动类型转换

数据类型按精度（容量）大小排序 很重要，需要背过

char → int → long → float → double

byte → short → int → long → float → double

自动数据类型转换细节

  1. 有多种类型的数据混合运算时，系统首先自动将所有数据类型转换成容量最大的那种数据类型，然后再进行运算。

  2. 当我们把精度（容量）大的数据类型赋值给精度（容量）小的数据类型会报错，反之则自动进行类型转换

  3. byte和short不会与char之间自动相互转换。

当把一个数值赋值给byte类时，先判断该数是否在byte范围内，若是则没问题。（这里不符合整数常量默认int类型规则）
例：
byte b1 = 10; //正常编译不报错

int n1 = 10;
byte b1 = n2; //报错，因为要将int类数值存储进精度较小的byte中

  1. byte、short、char三者可以运算，在运算时首先转换为int类型。
    例；
    byte b2 = 1;
    byte b3 = 2;
    short s1 = 1;
    byte b4 = b2 + s1; //如细节4所述，先转换成int再运算，byte接收int数据不会自动转换，编译报错

    byte b4 = b2 + b3; //两个同为byte类运算，也需要先自动转换为int类，所以用byte接收也会编译报错

  2. boolean类不参与自动转换
