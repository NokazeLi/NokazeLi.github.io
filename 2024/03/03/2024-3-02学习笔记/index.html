2024-3-02学习笔记

今天的内容是运算符

关系运算符（比较运算符）

关系运算符的结果是boolean类型，只能是true和false

关系表达式经常用在if结构的条件或者循环结构的条件中

==、!=、<、>、<=、>=、instanceof(检查是否是类的对象，到面向对象时会细讲)

关系运算符组成的表达式，称为关系表达式

逻辑运算符

用于连接多的条件（多个关系表达式），最终结果也是一个boolean值

&、&&、|、||、!、^（逻辑异或）

逻辑运算符规则
a & b:&叫逻辑与 a和b同时为true则结果为true否则为false。
a && b：短路与 a和b同时为true则结果为true否则为false。

a | b：逻辑或 当a和b中有一个为true则结果为true否则为false。
a || b：短路或 当a和b中有一个为true则结果为true否则为false。
!a:取反或者叫非运算。当a为true时结果为false，反之亦然。

a ^ b：逻辑异或，当a和b不同时结果为true否则为false。

&和&&的使用区别

  1. &&：如果第一个条件为false，则最终结果直接为false第二个条件不会判断，效率高。

  2. &：不管第一个条件是否为false，第二个都要判断，效率低。

  3. 开发中基本都是使用&&

|和||的使用区别

  1. ||：如果第一个条件为true，则最终结果直接为true第二个条件不会判断，效率高。

  2. |：不管第一个条件是否为true，第二个都要判断，效率低。

  3. 开发中基本都是使用||。

逻辑非与逻辑异或

!a:逻辑非或者逻辑取反，若a为true则!a为false，反之亦然。

a^b：逻辑异或，a和b的结果不同时，a^b为true；a和b结果相同时a^b为false。

一个练习(读代码分析过程和输出结果)

因为过程会有点长，在下面分析：
if((z++ == 46)&&(y = true))z++;
//z++ == 46,先比较后自增，所以为true，z为47；因为时&&程序进关系表达式2，y = true 注意这里是=赋值不是==，所以也是true。条件符合进入代码体 z++，z为48
if((x = false)||(++z == 49))z++;
//同样第一部分是赋值=不是==，x为false因为是||所以要判断条件2，++z == 49先自增后比较z为49且条件为true，进入代码体z++，z为50.
所以最终输出为50。

赋值运算符

将某个运算后的值，赋给指定变量，除了=之外还有+=、-=、*=、=

赋值运算符的特点

  1. 运算顺序从右往左（先运算完右边的整体再赋值给左边变量）

  2. 赋值运算符的左边只能是一个变量，右边可以是变量、表达式或者常量。

  3. 复合赋值运算符等价于以下效果
    a += 3; a = a + 3;

  4. 复合赋值运算符会进行类型转换

但是

byte b = b + 2; //这个会报错，因为右边是int型无法赋值给byte类变量b

b++; //底层也是包含一个强类型转换的

三元运算符

基本语法：
条件表达式 ? 表达式1 : 表达式2;

规则；

  1. 如果条件表达式为true，运算后的结果是表达式1。

  2. 如果条件表达式为false，运算后的结果是表达式2。

一个练习

分析过程（有个小trick）：

  1. a > b为false
  2. 返回b–，但是请注意因为是b–所以先赋值后自减，所以result为99，b为98

三元运算符细节

  1. 表达式1和表达式2要是可以赋值给左边变量的类型，或者可以自动转换的。

  2. 三元运算符可以转换为if else语句
