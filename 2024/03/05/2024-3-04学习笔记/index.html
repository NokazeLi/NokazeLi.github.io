2024-3-04学习笔记

今天主要学了位运算和程序控制结构的一部分，位运算很重要以前也没有学过，规则要背过需要重点留意。程序控制主要是if-else结构，之前掌握的较好。

位运算符

二进制（Binary）在运算中的说明

  1. 二进制是逢2进位制，0、1是基本算符。

  2. 现代的电子计算机技术全部采用的二进制，因为只使用0、1两个算符，非常简洁方便，易于用电子方式实现。

原码、反码、补码（重点 要背过）

  1. 二进制的最高位是符号位：0表示正数，1表示负数。

  2. 正数的原码、反码、补码都一样。（三码合一）

  3. 负数的反码 = 原码符号位不变，其他位取反。

  4. 负数的补码 = 反码 + 1，负数的反码 = 补码 - 1。

  5. 0的反码和补码都是0。

  6. Java没有无符号数，换言之Java中得数都是有符号的。

  7. 计算机运算得时候都是以补码得方式进行的。（所以自己算要先反 → 补 → 算 → 反 → 原）

  8. 看运算结果，要看原码。

位运算符

Java中有七个位运算符（&、|、^、~、>>、<<、>>>）

分两组归纳：

第一组：

  1. 按位与(&):两位全为1，则结果为1，否则为0。
    例:
    11011110 & 11000001 = 11000000
    11011110
    11000001 &
    ———————
    11000000

  2. 按位或(|)：两位有一个为1，则结果为1，否则为0。

来一个有负数的例子：
1 | -2 = -1
1的原码:00000000 00000000 00000000 00000001
因为1是正数，反码补码都和原码相同，故补码亦如上。

-2原码：10000000 00000000 00000000 00000010
反码：11111111 11111111 11111111 11111101
补码：11111111 11111111 11111111 11111110
补码运算：
00000000 00000000 00000000 00000001
11111111 11111111 11111111 11111110
——————————————————
11111111 11111111 11111111 11111111
此时结果为补码，应先转换成原码。
反码：1111111 11111111 11111111 11111110
原码: 1000000 00000000 00000000 00000001
3. 按位异或(^)；两位一个为0一个为1时，则结果为1，否则为0.

  1. 按位取反(~)： 0 -> 1, 1 -> 0

第二组：

  1. 算术右移(>>)：低位溢出（就是舍弃的意思），符号位不变，并用符号位补齐溢出的高位。
    例：
    1 >> 2 = 0

    00000000 00000000 00000000 00000001 >> 2
    00000000 00000000 00000000 00000000

  2. 算术左移(<<):符号位不变，低位补0。
    例：
    1 << 2 = 4

    00000000 00000000 00000000 00000001 << 2
    00000000 00000000 00000000 00000100

  3. 逻辑右移(>>>):也叫无符号右移，低位溢出高位补0，符号位也要移动

  4. 特别说明:不存在<<<。

程序结构

        因为之前学的比较多，笔记记录没有非常详细，如果有问题查询相应教程
  1. 顺序结构
  2. 分支结构
  3. 循环结构

顺序结构介绍：

程序从上到下逐行地执行，中间没有任何判断和跳转。

Java中定义变量时采用合法的前向引用。

分支控制 if-else 介绍：

  1. 单分支
    if(条件表达式){
    执行代码块;
    }

  2. 双分支
    if(条件表达式){
    执行代码块1;
    }else{
    执行代码块2；
    }

  3. 多分支
    if(条件表达式1){
    执行代码块1;
    }else if(条件表达式2){
    执行代码块2;
    }
    ….
    else{
    执行代码块n;
    }
